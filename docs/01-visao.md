# Visão e Missão

## O Problema

O trader varejista brasileiro enfrenta desafios significativos:

- **Informação fragmentada**: Dados de mercado espalhados em múltiplas plataformas
- **Custos elevados**: Ferramentas profissionais com preços proibitivos
- **Complexidade técnica**: Análise SMC/ICT requer estudo extensivo
- **Latência**: Dados atrasados ou indisponíveis em momentos críticos
- **Ruído**: Excesso de indicadores sem clareza de ação

## Público-Alvo

### Persona Principal: Trader Varejista B3

- **Perfil**: Pessoa física que opera na B3 (ações, ETFs, FIIs)
- **Experiência**: Intermediário (1-5 anos de mercado)
- **Capital**: R$ 10.000 a R$ 500.000
- **Frequência**: Day trade e swing trade
- **Dor principal**: Identificar pontos de entrada com alta probabilidade

### Personas Secundárias

- **Investidor de longo prazo**: Busca timing para aportes
- **Trader iniciante**: Aprendendo análise técnica
- **Profissional autônomo**: Gestores de patrimônio independentes

## Princípios Orientadores

### 1. Simplicidade Primeiro

Cada funcionalidade deve reduzir complexidade, não adicionar. O trader deve entender o sinal em segundos.

### 2. Dados Confiáveis

Preferimos mostrar "indisponível" a exibir dados incorretos. A confiança é construída pela precisão.

### 3. Transparência Metodológica

O usuário deve entender *por que* um sinal foi gerado, não apenas *qual* é o sinal.

### 4. Responsabilidade Educacional

Não vendemos "dinheiro fácil". Todo sinal inclui riscos e gestão de posição.

### 5. Performance sobre Features

Melhor ter 3 funcionalidades rápidas que 10 lentas. Latência mata oportunidades.
